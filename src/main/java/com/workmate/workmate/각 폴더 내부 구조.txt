[기본 폴더 역할 정리]

1. controller
- 역할: 클라이언트 요청을 받는 입구
- 하는 일:
  - URL 매핑 (/login, /users 등)
  - Request 데이터 받기 (@RequestBody, @PathVariable 등)
  - Service 호출
  - 결과를 Response로 반환
- 특징:
  - 로직을 거의 작성하지 않음
  - 요청 → 서비스 연결만 담당

--------------------------------------------------

2. service
- 역할: 비즈니스 로직 처리 (핵심)
- 하는 일:
  - 데이터 검증
  - 조건 분기
  - 여러 Repository 호출
  - 실제 기능 구현 (로그인, 회원가입, 스케줄 생성 등)
- 특징:
  - 프로젝트의 중심 로직
  - Controller와 Repository 사이에서 중간 처리

--------------------------------------------------

3. repository
- 역할: DB와 직접 통신
- 하는 일:
  - 데이터 조회 (SELECT)
  - 저장 (INSERT)
  - 수정 (UPDATE)
  - 삭제 (DELETE)
- 특징:
  - JpaRepository 등을 상속해서 사용
  - SQL 대신 메서드 이름으로 쿼리 생성 가능

--------------------------------------------------

4. entity
- 역할: DB 테이블 구조 정의
- 하는 일:
  - 클래스 = 테이블
  - 필드 = 컬럼
  - @Entity, @Id 등의 어노테이션 사용
- 특징:
  - 실제 DB와 1:1 매핑되는 객체
  - 데이터를 저장하는 구조 자체

--------------------------------------------------

5. dto (Data Transfer Object)
- 역할: 데이터 전달용 객체
- 하는 일:
  - 요청 데이터 전달 (Request DTO)
  - 응답 데이터 전달 (Response DTO)
- 특징:
  - Entity를 그대로 노출하지 않기 위해 사용
  - 필요한 데이터만 담아서 전달

--------------------------------------------------

[전체 흐름]

클라이언트 요청
→ Controller
→ Service
→ Repository
→ DB

(DB 결과 반환)
→ Repository
→ Service
→ DTO 변환
→ Controller
→ 클라이언트 응답

--------------------------------------------------

[한 줄 요약]

controller = 요청 받는 곳
service = 실제 로직 처리
repository = DB 담당
entity = DB 구조
dto = 데이터 전달용 객체
